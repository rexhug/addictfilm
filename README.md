# 🎬 Кинотрекер — Telegram Mini App

Публичный личный кинотрекер: каждый пользователь ведёт свой список фильмов,
отмечает просмотренное, ставит оценки и видит собственную статистику.
Опционально можно подключить одного партнёра и смотреть только общие итоги
пары — без открытия личных списков друг другу.

## Что умеет

- Поиск через **kinopoisk.dev** с fallback на OMDb и Wikidata.
- Личные списки «Хочу», «Смотрел» и личный топ по оценкам.
- Оценки 1–10, отметка о просмотре и безопасное удаление только из своего списка.
- Личная статистика: экранное время, жанры, актёры, средняя оценка и итоги года.
- Пара по взаимному приглашению в Telegram: совместимость оценок, общие фильмы,
  жанры, актёры, совпадения и самые спорные фильмы.
- Личные данные приватны: партнёр не получает доступ к списку или оценкам другого
  человека вне агрегированной общей статистики.

## Архитектура

```
Telegram Mini App
  └─▶ frontend/ — ванильный JS, интерфейс и Telegram WebApp SDK
        └─▶ backend/ — FastAPI: авторизация initData и JSON API
              ├─ database.py — SQLite (WAL), общий каталог + личные списки
              ├─ auth.py — проверка подписи Telegram
              ├─ search.py — kinopoisk.dev → OMDb + Wikidata
              └─ kinopoisk.py / omdb.py / wikidata.py — клиенты источников
```

Каталог `movies` общий. Пользовательские данные лежат отдельно: `users`,
`user_movies`, `ratings`, `comments`. Связь пары хранится в `partnerships`, а
одноразовые ссылки — в `partner_invites`.

## Быстрый старт

1. Создать бота через @BotFather и настроить для него **Main Mini App**.
2. Получить токен kinopoisk.dev у @kinopoiskdev_bot.
3. Скопировать настройки и заполнить секреты:

   ```bash
   cp .env.example .env
   ```

   `BOT_USERNAME` нужен для ссылки приглашения партнёра. Его указывают без `@`.
4. Установить зависимости и запустить приложение:

   ```bash
   python3 -m venv .venv
   .venv/bin/pip install -r requirements.txt
   cd backend && ../.venv/bin/uvicorn main:app --port 8077
   ```
5. Для разработки открыть HTTPS-туннель:

   ```bash
   cloudflared tunnel --url http://localhost:8077
   ```

6. В @BotFather указать URL туннеля как Main Mini App или кнопку меню.

## Данные и миграции

- Перед изменением схемы запускается `database.backup_db()` — это консистентный
  бэкап через `VACUUM INTO`; хранятся последние 7 копий.
- Текущая тестовая БД уже мигрирована на многопользовательскую схему без удаления
  старых таблиц.
- История из прежнего `movie_bot` перенесена отдельным идемпотентным скриптом:

  ```bash
  .venv/bin/python scripts/import_movie_bot.py
  ```

  Скрипт читает источник только в режиме read-only, берёт два Telegram ID из его
  `.env`, делает отдельный бэкап Mini App и переносит общий старый список в
  личные списки обоих пользователей. Оценки и комментарии остаются личными.
  Служебные идентификаторы старых сообщений Telegram не переносятся: Mini App
  ими не пользуется.

## Проверки

```bash
.venv/bin/python -m unittest discover -s backend/tests -v
PYTHONDONTWRITEBYTECODE=1 .venv/bin/python -m compileall -q backend
node --check frontend/app.js
```
