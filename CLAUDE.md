# Movie Mini App — инструкции для агента

**ПУБЛИЧНЫЙ Telegram Mini App** для поиска, просмотра и оценивания фильмов
(модель Кинопоиск/Letterboxd, single-user). Любой пользователь Telegram
регистрируется при первом входе; у каждого свой список и оценки, каталог
фильмов общий, community-рейтинг = средняя оценка всех пользователей.

> ⚠️ Курс сменился (2026-07): раньше это было приложение «на двоих» (Денис +
> Котятко). Теперь — публичный продукт с мультитенантностью. Любые старые
> упоминания «пользователей всегда двое» — устарели.

## Жёсткие правила
- **Весь UI — на русском**. Комментарии в коде тоже русские.
- **TMDb не предлагать**: у Дениса не приходит письмо верификации, ключ получить
  невозможно. Основной источник — kinopoisk.dev (токен через @kinopoiskdev_bot).
- **Мультитенантность**: пользователей МНОГО, данные изолируются по `user_id`.
  Схема: `users` (любой TG-юзер) / `films` (общий каталог, dedup по imdb_id) /
  `user_films` (per-user статус/оценка/коммент). Никакого белого списка —
  регистрируется любой с валидной подписью initData.
- **Лимит kinopoisk.dev** ~200 req/сутки на free → публику это не выдержит.
  Стратегия: кэш-каталог `films` (фильм тянем из API один раз), позже —
  cache-first поиск + rate-limit + возможно платный тариф.
- Данные бережём: перед миграциями схемы — бэкап (`database.backup_db`).
- Секреты только в `.env` (gitignored).

## Стек и запуск
- Backend: FastAPI + aiosqlite (WAL), из `backend/`: `../.venv/bin/uvicorn main:app --port 8077`
- Frontend: ванильный JS + Telegram WebApp SDK, тема из `--tg-theme-*`.
- Авторизация: initData HMAC (`backend/auth.py`), заголовок `X-Init-Data`.
- Публичный HTTPS для Telegram: `cloudflared tunnel --url http://localhost:8077`
  (прототип) или Cloudflare Tunnel с доменом (постоянно).
- venv уже создан: `.venv/` (fastapi, uvicorn, aiosqlite, aiohttp, python-dotenv).

## Состояние
**Фазы A и B — сделаны и проверены** (боевые токены в `.env`).
- A (мультитенантный фундамент): схема users/films/user_films, авторизация-upsert
  без whitelist, per-user эндпоинты, community-рейтинг, персональная статистика.
- B (защита лимита kinopoisk): `backend/ratelimit.py` — кэш поисковых запросов
  (`search.cached_search`), дневной бюджет внешних вызовов, per-user throttle (429).
- C (discovery): `/api/browse` (популярное/топ/жанры) + `/api/genres`; вкладка
  «🌐 Каталог» (стартовый экран), карточка фильма с community-скором.
- D (статистика с графиками): KPI-плитки, гистограмма оценок, бары жанров/
  актёров/режиссёров, «Итоги года» — CSS-графики, single-hue из темы Telegram.

**Все 4 фазы плана (A-D) сделаны** + UX-полировка + премиум-редизайн (тёмная
iOS-система, i18n RU/EN) + импорт данных старого бота + **Фаза E «Пара»**
(опциональный слой: инвайт через Telegram deep-link, совместимость, общая
статистика; личные списки остаются приватными — пара НЕ обязательна).
