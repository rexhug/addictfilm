# Уроки и грабли movie_bot (обязательно к прочтению)

Всё, что было добыто потом и кровью при разработке бота. Не наступать повторно.

## Источники данных

### kinopoisk.dev — основной источник ⭐
- **Хост**: `https://api.poiskkino.dev/v1.4` (домен kinopoisk.dev → 301 на poiskkino.dev,
  видимо юр. давление Яндекса). Если API «умер» — первым делом проверить актуальный хост.
- **Токен**: выдаёт Telegram-бот **@kinopoiskdev_bot** — без email-верификации, мгновенно.
- **Лимит**: ~200 запросов/сутки на бесплатном тарифе → обязателен in-memory кэш
  (в `kinopoisk.py` уже есть) и `selectFields` (меньше трафика).
- **ГРАБЛИ selectFields**: `selectFields=persons` работает, а `selectFields=persons.name` —
  возвращает ПУСТО. Вложенные поля персон нельзя выбирать точечно.
- **Кириллица в query**: обязательно URL-encode (aiohttp params делает сам; curl — `--data-urlencode`).
- Один запрос `/movie/search` даёт сразу: рус. название, рус. постер (Яндекс CDN 600×900),
  rating.kp + rating.imdb, жанры по-русски, описание, персон, externalId.imdb.
- Богатые поля `/movie/{id}`: similarMovies, sequelsAndPrequels, top250, slogan,
  ageRating, watchability, facts — задел на фичи.
- `externalId.imdb` используем как наш `imdb_id` (дедупликация); если его нет —
  синтетический `kp_<id>`.
- Батч-запросы по many imdb_id: `GET /movie?externalId.imdb=tt1&externalId.imdb=tt2…` —
  так делали бекфилы рейтингов/кастов.

### TMDb — НЕ ИСПОЛЬЗОВАТЬ
Письмо верификации аккаунта **не приходит неделями** (несколько аккаунтов, Gmail).
Ключ получить невозможно. Не предлагать снова.

### Wikidata — официальные русские названия (fallback)
- **ГРАБЛИ 403**: Wikimedia режет запросы без контакта в User-Agent. UA обязан быть вида
  `MovieBot/1.0 (https://github.com/...; contact) python-aiohttp` — иначе HTTP 403,
  причём на одних машинах работает, на других нет (интермиттентно!).
- SPARQL по списку IMDb ID одним запросом: `?item wdt:P345 ?imdbId; rdfs:label ?label`
  с FILTER(LANG='ru') — см. `wikidata.get_titles_by_imdb`.
- **Правило кириллицы**: название из Wikidata применять только если оно кириллицей,
  иначе «Apex» затрёт хорошее «Апекс» (см. `search.best_title`).

### OMDb — надёжный fallback
- Ключ уже есть (в movie_bot/.env). 1000 req/день.
- Поиск англоязычный сильный: «Hangover» отдаёт всю франшизу → на этом построен
  доиск сиквелов (`search._expand` + `_is_sequel`: только Part II/2/3, без
  одноимённого мусора — ремейков/документалок).
- Русские названия частей франшиз РАЗНЫЕ («Мальчишник в Вегасе» / «Мальчишник: Часть III»),
  поэтому русский поиск находит не всё — доиск через английское название обязателен.

## База данных
- SQLite + **WAL** (`PRAGMA journal_mode=WAL`, `synchronous=NORMAL`) — двое могут
  жать кнопки одновременно без «database is locked». WAL прописывается в файл навсегда.
- В .gitignore обязательно `*.db-wal` и `*.db-shm`.
- Бэкап: `VACUUM INTO` — консистентная копия на горячей базе (см. `database.backup_db`,
  ротация последних 7). В боте крутится ежедневно в 05:00.
- Миграции — простые `ALTER TABLE ... ADD COLUMN` в try/except в init_db().
- N+1 в списках лечится `get_ratings_bulk` (одним IN-запросом).

## Логика (выстраданные правила)
- **Ничьи — честно**: «Лучший фильм» при пяти 10/10 — это ложь; показываем всех.
  «Актёр года» — только при единоличном лидерстве.
- **Режим ввода один**: любые состояния «жду текст» взаимоисключающие, и кнопки
  меню всегда сбрасывают их (иначе текст кнопки сохранится как комментарий).
- **Тап по оценке = просмотрено**: не терять оценку из-за рассинхрона статуса.
- **Никаких тихих return**: каждое сохранение — в лог; необработанный callback —
  ловить и сообщать пользователю (иначе «нажал и ничего» недебажимо).
- **Старые кнопки в чате**: при смене формата callback-токенов старые сообщения
  остаются — парсить толерантно и отвечать «кнопка устарела».
- Даты храним ISO (UTC), показываем по-человечески («29 июня») — `ru.human_date`.
- **Мультитенантность** (курс сменился 2026-07, было «на двоих»): данные
  изолируются по `user_id`. Каталог `films` — общий (фильм тянем из API один
  раз, dedup по imdb_id), состояние — в `user_films` (статус/оценка/коммент на
  юзера). Оценка автоматически добавляет фильм в список и ставит «просмотрено».
- «Ничьи — честно» переносится на личный топ: при равных оценках показываем всех,
  «актёр года» — только при единоличном лидерстве.

## Telegram
- httpx логирует каждый getUpdates → лог 52МБ/месяц. Глушить:
  `logging.getLogger("httpx").setLevel(logging.WARNING)`.
- PicklePersistence не сохраняет job_queue → одноразовые напоминания умирают
  при рестарте; надёжно только `run_daily` (пересоздаётся на старте).
- Mini App: initData проверяется HMAC-SHA256 (secret = HMAC("WebAppData", bot_token)) —
  см. `backend/auth.py`. Публичный продукт: белого списка НЕТ, пускаем любого с
  валидной подписью и регистрируем через `db.upsert_user` (`current_user` в main.py).
- UI-язык проекта — русский. Код-комментарии тоже русские.
